# Zotero Translate

An **LLM-first** translation plugin for Zotero 7. Translate PDF selections, item
titles, abstracts, annotations and notes using large language models — with a
few free, no-API-key engines kept as zero-config fallbacks.

> This project is a derivative of
> [windingwind/zotero-pdf-translate](https://github.com/windingwind/zotero-pdf-translate)
> ("Translate for Zotero"), licensed under **AGPL-3.0**. It re-focuses that
> plugin on LLM translation, adds a set of OpenAI-compatible providers, and
> removes the API-key commercial MT and self-hosted engines. See
> [Differences from upstream](#differences-from-upstream).

## Translation engines

### LLM engines (primary)

These are sorted to the top of the engine list. Each needs an API key
(Settings → Translate → set the secret) and exposes editable endpoint, model,
temperature, prompt, streaming and custom-params settings.

- **ChatGPT** / **Azure OpenAI** / 3 **Custom GPT** slots
- **Claude** (Anthropic)
- **Gemini** (Google)
- OpenAI-compatible presets: **DeepSeek**, **Aliyun Bailian (Qwen)**,
  **MiniMax**, **Moonshot (Kimi)**, **Zhipu GLM**, **Volcengine Ark (Doubao)**,
  **SiliconFlow**, **OpenRouter**

Any OpenAI chat-completions-compatible provider not listed can be used through a
**Custom GPT** slot by setting its endpoint and model.

### Free no-key engines (fallback)

No API key, no configuration — used as the out-of-the-box default:

- Sentence: Google, Google(API), Bing, Youdao, Haici, Tencent Transmart,
  Volcengine Web, CNKI
- Dictionary: Bing Dict, Cambridge, Collins, FreeDictionaryAPI, Haici, Weblio,
  Youdao Dict

## Adding an OpenAI-compatible provider

Presets are generated by a single factory in
`src/modules/services/gpt.ts` (`createOpenAICompatibleService`) that reuses the
shared streaming core `gptTranslate`. To add a provider:

1. Add an entry to `openAICompatiblePresets` (and the `PresetID` union) in `gpt.ts`.
2. Add its default `<id>.endPoint` / `.model` / `.temperature` / `.prompt` /
   `.stream` / `.customParams` prefs in `addon/prefs.js`.
3. Add a `service-<id>=DisplayName` line in `addon/locale/*/addon.ftl`.
4. Register it in `src/modules/services/index.ts`.

## Differences from upstream

- LLM engines sort first and are the focus; free no-key engines remain as
  zero-config fallbacks.
- Added OpenAI-compatible presets (DeepSeek, Qwen, MiniMax, Moonshot, Zhipu,
  Doubao, SiliconFlow, OpenRouter).
- Removed all API-key commercial MT engines (Baidu, Tencent, DeepL Pro,
  Microsoft, NiuTrans, Caiyun, etc.) and self-hosted engines (DeepLX,
  MTranServer, NLLB, Pot, LibreTranslate).
- New plugin identity (`zotero-translate@niehu.github.io`) so it coexists with
  the original plugin.

## Development

